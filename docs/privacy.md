# Privacy Audit

## Overview

The Privacy Audit page (`/privacy`) helps you understand what telemetry and diagnostic data Claude Code collects. It provides tools to inspect telemetry files, review MCP server authentication status, locate data storage paths, and follow best practices for privacy.

## Features

### Telemetry File Inspector
Browse and inspect the telemetry data files that Claude Code generates. Each file is displayed with its contents in a readable, formatted view. Fields are annotated to explain what each data point represents (e.g., session duration, commands executed, error counts).

### MCP Auth Status
View the authentication state of all configured MCP (Model Context Protocol) servers. For each server, see:
- Server name and endpoint URL
- Authentication method (API key, OAuth, none)
- Whether credentials are stored locally
- Last authentication timestamp
- Token expiration status

### Data Locations Reference
A reference table listing every directory and file where Claude Code stores data on your machine:
- `~/.claude/` -- Main configuration directory
- `~/.claude/projects/` -- Per-project conversation history
- `~/.claude/logs/` -- Debug log files
- `~/.claude/paste-cache/` -- Cached paste content
- `~/.claude/statsig/` -- Feature flag cache
- `~/.claude/telemetry/` -- Telemetry event files
- `~/.claude/credentials.json` -- Authentication credentials
- `CLAUDE.md` files -- Project-level memory

### Privacy Tips
A curated list of actionable recommendations for managing your privacy:
- How to disable telemetry collection
- How to clear stored conversation history
- How to revoke MCP server tokens
- How to audit what data leaves your machine
- How to configure Claude Code for air-gapped environments
- Best practices for sensitive projects

## API Endpoints

### GET /api/telemetry
Returns parsed telemetry data from Claude Code's telemetry directory. Response includes:
- List of telemetry files with metadata (filename, size, last modified)
- Parsed event data from each file (event type, timestamp, payload)
- Summary statistics (total events, event types, date range)

Query parameters:
- `file` -- Return data from a specific telemetry file.
- `limit` -- Maximum number of events to return (default: 200).

### GET /api/mcp-auth
Returns the authentication status of configured MCP servers. Response includes:
- Array of server objects with connection details and auth status
- Credential storage locations
- Token validity information

No query parameters. Returns all configured servers.

## How It Works

1. **Telemetry Inspection**: The backend reads files from `~/.claude/telemetry/` (or the platform-appropriate telemetry directory). Each file contains JSON-formatted telemetry events. The backend parses these into structured objects and annotates fields with human-readable descriptions.

2. **MCP Auth Audit**: The backend reads MCP server configuration from Claude Code's config files and checks for stored credentials (API keys, OAuth tokens). It reports the authentication method and whether tokens are expired or still valid, without exposing the actual credential values.

3. **Data Locations**: The reference table is generated by checking which Claude Code data directories exist on the current machine. Directories that exist are marked as active, with file counts and total sizes. Directories that do not exist are listed but marked as inactive.

4. **Privacy Tips**: Static content curated from Claude Code documentation and best practices. Updated with the dashboard release.

## Troubleshooting

### Telemetry directory is empty
- Telemetry collection may be disabled in your Claude Code configuration. This is expected if you have opted out.
- Check that the telemetry path matches your operating system's conventions.

### MCP auth shows "unknown" status
- The MCP server may use a non-standard authentication method that the dashboard does not recognize.
- Ensure MCP server configuration files are accessible to the dashboard backend.

### Data locations show incorrect paths
- Paths are based on default Claude Code installation directories. Custom installations may use different paths.
- Set the `CLAUDE_HOME` environment variable if your Claude Code data is in a non-standard location.

### Telemetry events appear unreadable
- Some telemetry events use binary or compressed formats in newer Claude Code versions.
- Update the dashboard to the latest version for improved parser support.

### Cannot disable telemetry
- Follow Claude Code's official documentation for opting out of telemetry. The dashboard displays data but does not control collection settings.
- Check `~/.claude/config.json` for telemetry-related settings.

### MCP credentials concern
- The dashboard never displays raw credential values. It only reports whether credentials exist and their validity status.
- To revoke credentials, use the MCP server's management interface or delete the relevant entries from Claude Code's credential store.
