Data Locations
~/.claude/
Root configuration directory for Claude Code. Contains settings, credentials, and project data.
~/.claude/projects/
Per-project conversation history, CLAUDE.md files, and project-specific settings.
~/.claude/telemetry/
Usage telemetry and event logs. Tracks tool usage, token counts, errors, and session data.
~/.claude/credentials.json
Authentication tokens and API keys for Claude and connected services.
~/.claude/settings.json
Global user preferences, model settings, and feature flags.
~/.claude/statsig/
Feature flag and A/B testing data from Statsig SDK. Determines which features are enabled.
~/.claude/mcp.json
MCP server configurations including command paths, arguments, and environment variables.
~/.claude/paste/
Cached paste content from clipboard operations during conversations.
Privacy Tips
Clear Telemetry Data
Remove all telemetry files by deleting the telemetry directory:
rm -rf ~/.claude/telemetry/
Claude Code will recreate the directory on next launch but existing data will be gone.
Disable Telemetry
Set the environment variable to opt out of telemetry collection:
export CLAUDE_CODE_DISABLE_TELEMETRY=1
Add this to your shell profile (~/.bashrc, ~/.zshrc) to make it permanent.
What Stays Local
Conversation history, project files, paste cache, MCP configs, and local settings all stay on your machine in ~/.claude/. They are never uploaded unless you explicitly share them.
What Gets Sent
Conversation messages are sent to Anthropic's API for processing. Telemetry events (tool usage, errors, token counts) may be sent for analytics. Statsig feature flag checks send anonymous identifiers.
Audit MCP Servers
MCP servers can access files and run commands on your machine. Review ~/.claude/mcp.json and project-level .mcp.json files regularly to ensure only trusted servers are configured.
